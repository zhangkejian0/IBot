# XBot 功能说明书

> 版本：1.0.0 · 最后更新：2026-06-16
> 平台：Android（minSdk 24）

XBot 是一款基于设备摄像头的实时 AI 感知应用，核心能力包括**人脸表情识别、手势识别、身份识别**，并配有一个可交互的**虚拟宠物（表情系统）**界面。应用采用横屏全屏交互，识别与虚拟形象共用同一套检测管线。

---

## 一、主界面与显示模式

主界面有两种互斥的显示模式，由「设置 → 显示模式 → 调试模式」开关控制：

| 模式 | 调试开关 | 显示内容 |
|---|---|---|
| **虚拟宠物模式（默认）** | 关闭 | 全屏虚拟人物网页（表情系统），人物会跟随用户的人脸位置注视，并根据表情切换情绪状态 |
| **调试模式** | 开启 | 摄像头实时预览 + 识别结果覆盖层（关键点、人脸框、手势骨架、各类标签）+ 状态面板 |

> 不论处于哪种显示模式，**摄像头取流与全部识别（表情/手势/身份）始终在后台运行**。显示模式仅决定把识别结果「画到屏幕上」还是「驱动虚拟形象」。

右上角始终提供「设置」入口（齿轮图标）。

---

## 二、核心识别能力

以下识别能力由独立的检测引擎驱动，每帧并行执行：

### 1. 人脸检测与表情识别
- **多脸检测**：基于 Google ML Kit，最多同时检测 3 张人脸，按面积排序，主脸（面积最大）携带完整表情与关键点。
- **人脸关键点**：基于 MediaPipe Face Landmarker，输出 478 个关键点 + 虹膜点，用于精细网格绘制。
- **表情分类**：基于 ARKit 52 blendshapes 规则式分类，可识别 **7 种表情**：

| 表情 | emoji | 含义 |
|---|---|---|
| 中性 neutral | 😐 | 无强烈情绪 |
| 高兴 happy | 😄 | 笑（嘴角上扬 + 脸颊上提） |
| 伤心 sad | 😢 | 嘴角下垂 + 八字眉 |
| 惊讶 surprised | 😲 | 张嘴 + 眉毛上抬 + 睁大眼 |
| 愤怒 angry | 😠 | 皱眉 + 抿嘴/眯眼 |
| 厌恶 disgusted | 🤢 | 皱鼻 + 上唇上提 |
| 恐惧 fearful | 😨 | 睁大眼 + 眉内抬 + 嘴角横向拉伸 |

### 2. 身份识别（认识我）
- 基于人脸特征向量（embedding）与已录入人物库的比对，给出最匹配的人物姓名与相似度。
- 多人脸时为每张脸维持稳定的「位置身份槽」，按位置追踪身份，避免逐帧串脸。
- 节流策略：每 1.2 秒重算一次身份，TTL 3 秒，保证连续性。
- 前提：需在「认识我」中录入人物后才能识别。模型未加载时显示「未加载模型」。

### 3. 手势识别
- 基于 MediaPipe Hand Detection，输出 21 点手部骨架与手势判定，支持多只手。

## 三、虚拟宠物（表情系统）

虚拟宠物是一个内嵌的网页应用（React + SVG），通过本地静态文件服务加载，运行在 WebView 中。它具备：

### 1. 多种显示风格
通过加载 URL 的查询参数切换：
- `?style=ambient`（**当前默认**）：暗背景 + 环境光晕 + 卡哇伊珍珠眼 + 底部状态文字。
- `?style=neon`：霓虹/LED 机器人脸，强外辉光，颜色随状态变化。
- （留空）：默认写实风格脸。

### 2. FSM 状态机（9 个状态）
虚拟人物有一个有限状态机，不同状态对应不同的表情、配色与微动：

| 状态 | 语义 |
|---|---|
| idle | 待机（中性） |
| gazing | 注视 |
| listening | 聆听 |
| thinking | 思考 |
| happy | 高兴（大笑） |
| confused | 困惑（歪头） |
| sleepy | 困倦（耷拉眼） |
| sleeping | 睡眠（闭眼） |
| waking | 苏醒 |

### 3. 情绪驱动（表情 → FSM 映射）
检测到的真人表情会映射为虚拟人物的状态（方案 A）：

| 真人表情 | → 虚拟人物状态 |
|---|---|
| 高兴 | happy |
| 中性 | idle |
| 伤心 | sleepy |
| 愤怒 / 厌恶 / 恐惧 / 惊讶 | confused |

为避免表情抖动导致状态频繁跳变，同一状态至少保持 600ms 才允许切换。

### 4. 注视跟随
虚拟人物的瞳孔会**跟随用户主脸在画面中的相对位置**移动，营造「始终看着主人」的效果：
- 主脸包围盒中心在画面中的归一化位置 → 瞳孔偏移方向（-1..1）。
- 前置摄像头水平镜像，与自拍预览方向一致。
- 注视信号绕过网页弹簧系统直接应用，跟随接近实时（零延迟平滑）。

## 五、技术架构概览

### 检测管线
```
摄像头取流（YUV420）
  → 每帧分发：
      ├─ MediaPipe Face（478点 + blendshapes → 表情分类）
      ├─ ML Kit（多人脸框）
      ├─ MediaPipe Hand（手势骨架）
      └─ 身份识别（节流：裁剪 → embedding → 比对人物库）
  → 聚合为 DetectionResult → 驱动覆盖层 / 虚拟形象
```

### 虚拟形象数据流
```
主脸表情 → EmotionMapper → FSM 状态
主脸位置 → 注视方向（镜像+零延迟）
              ↓（JS 桥接，节流 ~20fps，合并单次调用）
   WebView: window.__face.setState(...) / setGazeTarget(...)
              ↓
   网页 React 渲染（ambient 风格）
```

### 关键依赖
| 库 | 用途 |
|---|---|
| camera | 摄像头取流 |
| google_mlkit_face_detection | 多人脸检测 |
| kwon_mediapipe_landmarker | 人脸 478 点 + blendshapes |
| hand_detection | 手势识别 |
| tflite_flutter | 表情分类模型 |
| webview_flutter | 虚拟宠物网页 |

## 六、权限需求

| 权限 | 用途 |
|---|---|
| 摄像头（CAMERA） | 实时识别（必需） |
| 网络（INTERNET） | 本地静态文件服务（虚拟宠物，localhost） |

> 应用允许明文 HTTP 流量（`usesCleartextTraffic`），仅用于 WebView 访问本地 `localhost` 静态服务，不涉及外网。
